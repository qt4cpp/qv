# 変更仕様書: ハイブリッド 4 分割表示（macOS クラッシュ回避）

## 文書情報
- 作成日: 2026-02-08
- 対象プロジェクト: QV - DICOM Viewer
- 環境: macOS Tahoe / Python 3.13.5 / VTK 9.5.2 / PySide6 6.11 / ARM64

---

## 1. 背景 / 問題

### 1.1 現象
- macOS 環境で複数の `QVTKRenderWindowInteractor` を生成すると、
  `Render()` / `Initialize()` 時に SIGSEGV が発生する。
- VolumeViewer 単体でも `Render()` 呼び出しでクラッシュすることがある。
- ソフトウェア OpenGL、CPU Volume Mapper、`QSurfaceFormat(samples=0)` いずれも改善なし。

### 1.2 現状の安定構成
- **SliceViewerWidget**: `vtkImageReslice` → NumPy → `QImage` → `QLabel` 表示。
  OpenGL / RenderWindow 不使用。**安定動作中**。
- **VolumeViewer**: `Render()` を遅延実行する構成で起動は可能だが、描画自体は不安定。
  `_skip_interactor_init = True` + `enable_rendering()` による手動制御で運用中。

### 1.3 前回仕様の問題点
- 4 renderer viewport 分割（単一 RenderWindow に 4 つの vtkRenderer）方式は、
  安定動作している QLabel 方式の MPR を VTK レンダリングに戻すリスクがある。
- viewport 分割ではインタラクションの面ごと分離が困難（VTK interactor は RenderWindow 単位）。
- `Render()` 自体がクラッシュの本体であるため、コンテキスト数削減だけでは根本解決しない可能性がある。

---

## 2. 目的

1. macOS 環境での**起動・表示・操作時のクラッシュを回避**する。
2. Axial / Sagittal / Coronal / 3D Volume を **4 分割レイアウト**で同時表示する。
3. **安定動作している QLabel 方式の MPR を維持**し、リスクを最小化する。
4. OpenGL コンテキストを **Volume 3D 用の 1 つのみ**に限定する。

---

## 3. 非目標

- MPR 面での VTK ネイティブレンダリング復帰（将来 VTK バージョン更新時に再検討）。
- MPR 面のクロスヘア同期、Oblique スライス対応。
- 画質改善・新機能追加。
- Volume 3D の `Render()` クラッシュ自体の根本解決（本仕様は「構成の安定化」が主眼）。

---

```
def setup_panels(self) -> None:
    self._grid = QGridLayout(self)
    self._grid.setContentsMargins(2, 2, 2, 2)
    self._grid.setSpacing(2)

    # MPR ビューワ (QLabel 方式、OpenGL 不使用)
    self._axial_slot = ViewerSlot()
    self._sagittal_slot = ViewerSlot()
    self._coronal_slot = ViewerSlot()

    self._axial_viewer = SliceViewerWidget(axis=SliceAxis.AXIAL)
    self._sagittal_viewer = SliceViewerWidget(axis=SliceAxis.SAGITTAL)
    self._coronal_viewer = SliceViewerWidget(axis=SliceAxis.CORONAL)

    # Volume ビューワ (唯一の QVTK)
    self._volume_slot = ViewerSlot()
    # VolumeViewer は外部から注入（MainWindow が所有）

    # レイアウト配置
    self._grid.addWidget(self._axial_slot, 0, 0)      # 左上
    self._grid.addWidget(self._sagittal_slot, 0, 1)    # 右上
    self._grid.addWidget(self._coronal_slot, 1, 0)     # 左下
    self._grid.addWidget(self._volume_slot, 1, 1)      # 右下

    # ViewerSlot にビューワを配置
    self._axial_slot.set_viewer(self._axial_viewer)
    self._sagittal_slot.set_viewer(self._sagittal_viewer)
    self._coronal_slot.set_viewer(self._coronal_viewer)
    self._volume_slot.set_viewer(self._volume_viewer)

    # MprController セットアップ
    self._mpr_controller = MprController()
    self._mpr_controller.set_volume_viewer(self._volume_viewer)
    self._axial_viewer.set_controller(self._mpr_controller)
    self._sagittal_viewer.set_controller(self._mpr_controller)
    self._coronal_viewer.set_controller(self._mpr_controller)
```

## 10. フォールバック計画（Plan B）

### 10.1 発動条件
- Task D 完了後、VolumeViewer の `Render()` が依然 SIGSEGV を起こす場合。

### 10.2 方式: オフスクリーンレンダリング → QLabel 表示
```
python
# VolumeViewer に追加するメソッド
def render_to_image(self) -> QImage | None:
    """オフスクリーンで描画し QImage として返す"""
    rw = self.vtk_widget.GetRenderWindow()
    rw.SetOffScreenRendering(True)
    rw.Render()

    w2i = vtk.vtkWindowToImageFilter()
    w2i.SetInput(rw)
    w2i.Update()

    img = w2i.GetOutput()
    dims = img.GetDimensions()
    arr = vtk_to_numpy(img.GetPointData().GetScalars())
    arr = arr.reshape(dims[1], dims[0], -1)
    arr = np.flipud(arr)  # VTK は bottom-up

    qimg = QImage(arr.data, dims[0], dims[1], QImage.Format_RGB888)
    rw.SetOffScreenRendering(False)
    return qimg
```
### 10.3 Plan B 適用時のアーキテクチャ変化

- VolumeViewer のレイアウトを `QLabel` ベースに切替
- タイマーで定期的に `render_to_image()` → `QLabel.setPixmap()` を実行
- インタラクション（回転・WW/WL）は Qt イベント → VTK カメラ操作 → 再描画のループ

### 10.4 Plan B の制約

- リアルタイムレスポンスは低下する（フレームレート 5–15 fps 程度を想定）
- GPU による直接表示と比較して画質は同等だがレイテンシは増加

## 14. まとめ

| 項目 | 方針 |
|------|------|
| MPR 3面 | **QLabel 方式を維持**（OpenGL 不使用、安定動作） |
| 3D Volume | **単独 QVTK**（唯一の OpenGL コンテキスト） |
| レイアウト | **Qt QGridLayout** で 2×2 分割 |
| インタラクション | MPR は Qt wheelEvent、3D は VTK interactor |
| データ共有 | `VolumeModel` + `MprController` の既存機構を活用 |
| フォールバック | Volume Render 不安定時はオフスクリーン → QLabel (Plan B) |
| 最大のリスク | Volume 3D の `Render()` が依然クラッシュする可能性 → Plan B で対応 |
